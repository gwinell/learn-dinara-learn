## Quest 2. Repo

### Что нужно сделать

Запустить `src/ai_initial_module.sh` и исправить ошибки (не модифицируя скрипт).

### Проблемы на master

1. **`ai_module_2.sh` нет прав на выполнение** — скрипт вызывает `./ai_module_2.sh` и падает с `Permission denied`
2. **`important_data_for_ai_module_2.txt` содержит `1, 2, 42`** вместо `1, 2, 3, 4, 5` — скрипт ожидает ровно 5 строк с числами 1-5

### Решение

```bash
# Исправить права
chmod +x src/ai_module_2.sh

# Исправить файл данных
cat > src/important_data_for_ai_module_2.txt << 'EOF'
1
2
3
4
5
EOF

# Проверить
bash src/ai_initial_module.sh
# Должно вывести: "МОДУЛЬ 2 успешно загружен!"
```

### Коммит

```bash
git add src/
git commit -m "fix: исправлены права на скрипт и данные модулей"
git push origin develop
```

## Quest 3. Dirs & Files

### Что нужно сделать

Исправить структуру конфигурационных файлов для `ai_door_management_module.sh`.

### Проблемы на master

1. **Папка называется `door_management_fi`** (обрезано), скрипт ищет `door_management_files`
2. **Все файлы (.conf, .log, door_map) в одной папке** — нет подпапок `door_configuration/`, `door_logs/`, `door_map/`
3. **Лишний файл `door_21.conf`** — должно быть door_1..door_16
4. **`ai_door_control.sh` нет прав на выполнение**
5. **Баг в скрипте:** проверка `$count -eq 34`, но реальное количество файлов 33 (16+16+1). Нужно добавить `door_17.conf`

### Решение

```bash
# 1. Переименовать папку
cd src
mv door_management_fi door_management_files

# 2. Создать структуру подпапок
mkdir -p door_management_files/door_configuration
mkdir -p door_management_files/door_logs
mkdir -p door_management_files/door_map

# 3. Переместить файлы
mv door_management_files/door_*.conf door_management_files/door_configuration/
mv door_management_files/door_*.log door_management_files/door_logs/
mv door_management_files/door_map_1.1 door_management_files/door_map/

# 4. Удалить лишний файл
rm door_management_files/door_configuration/door_21.conf

# 5. Добавить door_17.conf (из-за бага в скрипте — count должен быть 34)
cp door_management_files/door_configuration/door_1.conf door_management_files/door_configuration/door_17.conf

# 6. Исправить права
chmod +x ../ai_door_control.sh

# 7. Проверить структуру
ls door_management_files/door_configuration/ | wc -l  # 17
ls door_management_files/door_logs/ | wc -l             # 16
ls door_management_files/door_map/                      # door_map_1.1

# 8. Запустить скрипт
cd ..
bash src/ai_door_management_module.sh
# Должен запустить ai_door_control.sh в фоне

# 9. Убить фоновый процесс
pkill -f ai_door_control
```

### Запись команд в quest3.sh

```bash
cat > src/quest3.sh << 'EOF'
#!/bin/bash
cd src
mv door_management_fi door_management_files
mkdir -p door_management_files/door_configuration
mkdir -p door_management_files/door_logs
mkdir -p door_management_files/door_map
mv door_management_files/door_*.conf door_management_files/door_configuration/
mv door_management_files/door_*.log door_management_files/door_logs/
mv door_management_files/door_map_1.1 door_management_files/door_map/
rm -f door_management_files/door_configuration/door_21.conf
cp door_management_files/door_configuration/door_1.conf door_management_files/door_configuration/door_17.conf
chmod +x ../ai_door_control.sh
EOF
chmod +x src/quest3.sh
```

### Коммит

```bash
git add src/
git commit -m "fix: исправлена структура конфигурационных файлов дверей"
git push origin develop
```

## Quest 7. Branches

### Что нужно сделать

Создать ветки по инструкции из `src/git_for_human/readme.txt`.

### Инструкция

1. Вся работа на `develop`
2. `key-branch` от develop с `main.key`
3. Копия `main-2.key`, зафиксировать
4. `feature/3-key` от key-branch
5. `main-3.key` в feature/3-key, зафиксировать
6. Слить feature/3-key в key-branch
7. `release/1.0` от develop
8. Слить key-branch → develop → release/1.0

### Решение

```bash
# Убедиться что на develop
git checkout develop

# Создать key-branch
git checkout -b key-branch
cp src/ai_help/main.key src/ai_help/main-2.key
git add src/ai_help/main-2.key
git commit -m "key-branch: добавлена копия ключа main-2.key"

# Создать feature/3-key
git checkout -b feature/3-key
cp src/ai_help/main.key src/ai_help/main-3.key
git add src/ai_help/main-3.key
git commit -m "feature/3-key: добавлена копия ключа main-3.key"

# Слить feature/3-key в key-branch
git checkout key-branch
git merge feature/3-key

# Создать release/1.0 от develop
git checkout develop
git checkout -b release/1.0

# Слить key-branch в develop
git checkout develop
git merge key-branch

# Слить develop в release/1.0
git checkout release/1.0
git merge develop

# Вернуться на develop
git checkout develop
```

3. Каждое изменение логировать

### Решение

```bash
# Начальное состояние
SIZE=$(wc -c < src/history_of_vim.txt | tr -d ' ')
DATE=$(date "+%Y-%m-%d %H:%M:%S")
SHA=$(shasum -a 256 src/history_of_vim.txt | awk '{print $1}')
echo "src/history_of_vim.txt - $SIZE - $DATE - $SHA - sha256" >> src/files.log
```

## Bonus Quest 9. Script

### Что нужно сделать

Написать `src/edit.sh` для замены строк в файлах с логированием.

### Код edit.sh

```bash
#!/bin/bash
if [ $# -ne 3 ]; then
    echo "Usage: $0 <file_path> <search_string> <replace_string>"
    exit 1
fi

FILE_PATH="$1"
SEARCH="$2"
REPLACE="$3"

if [ ! -f "$FILE_PATH" ]; then
    echo "File not found: $FILE_PATH"
    exit 1
fi

sed -i '' "s|$SEARCH|$REPLACE|g" "$FILE_PATH"

SIZE=$(wc -c < "$FILE_PATH" | tr -d ' ')
DATE=$(date "+%Y-%m-%d %H:%M:%S")
SHA=$(shasum -a 256 "$FILE_PATH" | awk '{print $1}')

echo "$FILE_PATH - $SIZE - $DATE - $SHA - sha256" >> src/files.log
echo "Replaced '$SEARCH' with '$REPLACE' in $FILE_PATH"
```

### Коммит

```bash
git add src/
git commit -m "feat: добавлен скрипт edit.sh"
git push origin develop
```
